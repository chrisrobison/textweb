How It Works

1

Real browser renders the page

Headless Chromium executes all JavaScript, CSS, and dynamic content ‚Äî exactly like a real user sees it.

2

Extract positions & semantics

TextWeb reads every visible element's bounding rect, text content, and interactivity. Links, buttons, inputs ‚Äî all identified.

3

Map to a character grid

Pixel positions become row/column positions on a text grid. Spatial layout is preserved ‚Äî things that are next to each other on screen are next to each other in text.

4

Annotate interactive elements

Every clickable link, button, input, and dropdown gets a reference number like [0], [1]. Agents say "click 3" or "type 7 hello" to interact.

# Render any page $ textweb https://news.ycombinator.com # Interactive mode ‚Äî browse with commands $ textweb --interactive https://github.com textweb> click 3 textweb> type 7 search query textweb> scroll down textweb> refs # JSON output for piping to agents $ textweb --json https://example.com # Custom grid size $ textweb --cols 80 --rows 24 https://example.com
# Start the server $ textweb --serve 3000 # Navigate to a page $ curl -X POST http://localhost:3000/navigate \ -H 'Content-Type: application/json' \ -d '{"url": "https://example.com"}' # Click an element $ curl -X POST http://localhost:3000/click \ -d '{"ref": 3}' # Type into an input $ curl -X POST http://localhost:3000/type \ -d '{"ref": 7, "text": "hello world"}'
const { AgentBrowser } = require('textweb'); const browser = new AgentBrowser({ cols: 120 }); // Navigate and get the text grid const { view, elements, meta } = await browser.navigate('https://example.com'); console.log(view); // The text grid console.log(elements); // { 0: { selector, tag, text }, ... } // Interact await browser.click(3); // Click element [3] await browser.type(7, 'hello'); // Type into element [7] await browser.scroll('down'); // Scroll down await browser.close();
